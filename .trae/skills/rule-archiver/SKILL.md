# 核心流程
1. 提取：从对话中识别高频错误、非直观逻辑及技术决策。
2. 转化：提炼为简洁规则，按规范/技术栈/部署/通用经验分类。
3. 检索：读取 `.trae/rules` 目录，验证规则是否已存在或需新建文件。
4. 定义：新建文件需包含如下 Frontmatter 头部：
   ```yaml
   ---
   alwaysApply: false 
   globs: "pattern/*"
   description: "描述内容"
   ---
   ```
5. 更新：执行去重，在对应类别标题下追加 Markdown 列表（* [类别]: 内容）。

# 关键规则
- 排除代码：严禁录入具体代码段，仅保留逻辑干货。
- 原子化：保持规则单一职责，避免逻辑耦合。
- 精简性：删除修饰语，确保规则可直接被 AI 解析执行。
